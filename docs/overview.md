# Project Overview

## What Is This?

A web application for securely collecting evidence and sources from a distributed group of people. All sensitive data is end-to-end encrypted client-side — a compromised server reveals nothing about submission contents or submitter identities.

## Core Concepts

### Projects
Created by an infrastructure admin (authenticated via an env-level password, no data access). Each project has its own asymmetric keypair and a configurable submission form. The project private key is generated client-side by the first MODERATOR and never exists in plaintext on the server.

### Roles

| Role | Can do |
|---|---|
| **Infrastructure admin** | Create / delete projects, generate the first MODERATOR invite link. No access to submission data. |
| **MODERATOR** | View all submissions + decrypted submitter contact info. Invite new users. Promote submitters to MODERATORs. |
| **Submitter** | Fill out the project form and submit evidence. Can view their own past submissions only. |

### Submissions
Each submission is encrypted with a unique symmetric key. That key is stored encrypted twice: once for the project public key (so MODERATORs can read it) and once for the submitter's public key (so the submitter can read their own). MODERATORs gain access to submissions via an encrypted copy of the project private key stored in the database, encrypted with their own public key.

### Invite Links
Generated by MODERATORs (or the infrastructure admin for the first MODERATOR link). Signed by the generator's private key. Options: expiry date, max uses, one-time use, MODERATOR vs submitter role. Includes a QR code for easy mobile access.

### Cross-Device Access
A user can generate a link/QR code from their current device. The key bundle is encrypted with a user-chosen passphrase using PBKDF2 + AES-GCM and embedded in the URL `#fragment` (never sent to the server). Scanning/opening on a new device and entering the passphrase restores full access.

### Export / Import
Same mechanism as cross-device linking, but as a downloadable file. Contains only the user's private key (passphrase-protected). MODERATOR memberships are re-fetched from the server on import — no stale data in the export file.

## Threat Model

**Protected against server compromise:**
- Submission contents (encrypted, server holds only ciphertext)
- Submitter identities and contact info (encrypted with project public key)
- Who submitted what (symmetric keys encrypted per-user)

**Not protected against:**
- A compromised client device (the private key is in localStorage)
- An MODERATOR who is themselves malicious (they have legitimate decryption access)

## Form Fields

Each project defines its own submission form. Supported field types:
- **Text** — freeform text input
- **Select** — single choice from a predefined list
- **File upload** — one or more files, encrypted client-side before upload
